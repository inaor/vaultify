Detection Pipeline

Vaultify uses a sequenced multi-layer detection pipeline to identify plaintext credentials on your endpoint.

Pre-Validate
File type and path filtering. Only relevant files enter the pipeline. Browser caches, node_modules, and system directories are excluded.
Layer 2: Context
Scans assignment patterns (key=value) for compound keywords like api_key, password, secret_access. Runs first on lines with = or : operators.
Layer 1: Value Patterns
54 regex patterns with prefix fast-skip. Matches known token formats: AKIA, ghp_, xoxb-, sk-, and 50 more.
Post-Validate
Deduplication by SHA256, entropy computation, confidence scoring. When both layers agree on the same value, confidence is highest.
Entropy Scoring

Shannon entropy measures the randomness of a string in bits per character. Real secrets are generated randomly and have high entropy. Code identifiers, English words, and placeholders have low entropy.

Entropy Range Interpretation Example
4.0+High confidence — likely a real secretAKIA5REXAMPLE1234
3.0-4.0Moderate — could be a secret or structured dataghp_XkDemoToken9mRt
< 3.0Low — likely a code identifier or placeholderloadingSpinnerClassName

Entropy is computed using the standard Shannon formula: H = -sum(p * log2(p)) for each unique character. The result is shown in cyan in the Review table and Findings Explorer.

Security Model

Vaultify is designed as a zero-secrets-at-rest tool. Secret values are never written to disk.

  • No plaintext.json — secret values exist only in your source files and briefly in server memory during a scan
  • Line snippets are redacted before storage — REDACTED_BY_VAULTIFY replaces the matched value
  • WebSocket broadcasts strip Value from findings — the browser never receives raw secrets
  • Live extraction at apply time — when you Vaultify a secret, the value is re-read from the source file, not from cache
  • File permissions: 0o700 for directories, 0o600 for files (owner-only)
  • WebSocket origin validation — only localhost connections accepted
  • Server binds to 127.0.0.1 only — not accessible from the network
